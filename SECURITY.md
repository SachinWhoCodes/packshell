# Security

packshell is designed so the API, database, dashboard, and platform operators never receive plaintext `.env` contents.

## Encryption model

- `.env` content is encrypted locally with AES-256-GCM.
- Each project has a random 256-bit project key.
- Each user has a local RSA-OAEP-4096 keypair generated by the CLI.
- The project key is encrypted separately for each member with that member's public key.
- Private keys remain in `~/.packshell/keys` and are never uploaded.

## Trust boundaries

- Vercel API routes verify Firebase ID tokens and CLI tokens.
- Firestore client access is denied by default; all writes go through server-side checks.
- Audit logs store actors, actions, project/environment metadata, counts, checksums, and version ids.
- Secret values are not stored in logs or dashboard responses.

## Operational notes

- Removing a member blocks future pulls and removes their future project key access. It cannot erase `.env` files or keys already copied to their machine.
- Rotate real secrets after offboarding a developer from sensitive projects.
- Keep `FIREBASE_PRIVATE_KEY` and `FIREBASE_SERVICE_ACCOUNT_BASE64` only in Vercel environment variables.
- Add the first platform owner manually in Firestore at `/superAdmins/{firebaseUid}`.
